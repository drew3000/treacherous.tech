A little resource on bypassing online censorship

Yours Truly

Started a small resource page for people on how to deal with different kinds of censorship, that I'm calling The Detour. It's currently got some information around technology that can help during an Internet Shutdown, since that's been some focus of conversation over the last week following the military coup in Myanmar, and the shutdown that happened there. But internet shutdowns are on the rise, so wanted to start with some posts on that.

I'll be moving on to include some resources on other kinds of internet censorship and how to work around it. The goal overall is to keep it end-user focused, though may add a post on the bigger, more involved projects.

I'm a Standard Notes user, and recently noticed that it has a simple blogging function that publishes on Listed.to. This is a great way for simple, offline writing and editing and then pushing the results you're happy with to a live site. I'm very interested in these methods of no-database, secure workflow publishing models. There's very little need to touch the internet until you're ready to publish something, as opposed to typical the typical set up you'd see in Wordpress, Drupal, Django or others that lump the front-end and administration parts of your website in one, easily attackable location by default. It's not an incredibly sophisticated publishing tool, but really easy for anyone to start, and the model is incredibly attractive if you wanted to extend it to a small team who could submit, review edit and publish using Standard Notes already existing encrypted sharing functions. It's much better than leaving unfinished drafts on a public website, or having to keep persistent connections to a web server while doing the research, writing and editing.
